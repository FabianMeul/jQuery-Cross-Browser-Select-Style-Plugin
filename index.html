jQuery Cross Browser Select Style Plugin

TESTED IN: IE7, IE8, IE9, Chrome (Mac/PC), Safari (Mac/PC), FireFox 3.62 (PC), FireFox 7 (Mac), FireFox 8 (PC), Firefox Android Mobile, iOS4/iOS5, Android Eclair/Froyo/Gingerbread/Honeycomb/Ice Cream Sandwich

Option Descriptions className : Default Value='select' Added v1.0 Override of the select's wrapping class. classText : Default Value= 'text' Added v1.0 Override of the select's selected value container. classArrow : Default Value='arrow' Added v1.0 Override of the select's "arrow section" responsive : Default Value='false' Added v1.0 Boolean. When true the select width becomes a percent [(parent element's width / select width) * 100], this allows the select to scale proportionate to its parent element. If false (default), the select remains its initial width (even if defined by a percent within the selector's CSS) arrow : Default Value='true' Added v1.0 Boolean. To show or hide the arrow. arrowWidth : Default Value='20' Added v1.0 A value, integer or percent to define the width of the “arrow section" of the styled selector. Examples are 50, '50px', '25%' arrowContent : Default Value='null' Added v1.0 Define content to be populated in the “arrow section". This can be a character or HTML. Images are recommended to be defined within CSS of the “classArrow" option extraStyles : Default Value='true' Added v1.0 Adds 3 classes that toggle for triggered events. On focus adds a class (defined by the 'classFocus' option), this class is removed on blur. Once a selection occurs, the select has a class added to it (defined by the “classEnd" option) classFocus : Default Value='focused' Added v1.0 Override of the select's "focused value" classEnd : Default Value='selected' Added v1.0 Override of the select's "selected value" class, applied once select has a value that is not null/undefined/'' display : Default Value='inline-block' Added v1.1 Override of the select's display type (traditional W3 Standard display types)
